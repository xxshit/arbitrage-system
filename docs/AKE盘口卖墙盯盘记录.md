# AKE 盘口卖墙盯盘记录

## 用户新思路

AKE 在盘口上方出现重点卖墙/压力带：

- 0.0020 附近：约 1.1M AKE 卖单
- 0.0021 附近：约 1.2M AKE 卖单
- 0.0022 附近：约 1.0M AKE 卖单
- 重点压力带：0.0020–0.0023

用户的两个假设：

1. 如果主力主动把价格拉到 0.0020–0.0022 以上，说明愿意吃掉或逼退这部分压力，后续可能继续拉盘，把上方做空/卖压爆掉或逼走。
2. 如果价格一直站不上 0.0020，则可能在 0.0020 下方开始派发，随后转为下跌。

## 我的补充判断

盘口挂单不能直接等同真实空单，因为挂单可以撤，也可能是诱导盘口。但它可以作为“主力是否愿意吃墙”的观察线索。

因此 AKE 卖墙盯盘必须结合：

- 是否站上 0.0020；
- 是否进一步站上 0.0022；
- 突破时 CVD 是否继续上涨；
- 持仓是否没有同步塌掉；
- 成交量是否放大；
- 基差是否继续正向打开；
- 是否出现空头爆仓跟随。

## 盯盘规则

### LARK 推送降噪规则

AKE 盘口墙属于“方向确认前的观察信息”，不能因为正资费、正基差、普通多头结构就频繁推送。

从 2026-07-18 起，AKE LARK 推送只保留三类更明确的信号：

- 卖墙突破：价格站上 0.0022，且 CVD/持仓配合；
- 墙下派发：价格站不上 0.0020，且量能活跃、价格或 CVD 转弱；
- 出货警报：放量、资金费率转负、基差明显打开同时出现。

普通“正资费 + 正基差 + 合约量大”的多头观察，只在网页里显示，不再自动推送 LARK。

同一个 AKE 思路推送增加 12 小时总冷却，除非后续单独设计更严格的“重大反转”强提醒。

### 卖墙突破 / 逼空观察

触发倾向：

- AKE 价格站上 0.0022；
- CVD 至少两个观察窗口为正；
- 持仓至少一个观察窗口没有走弱；
- 0.0020–0.0023 压力带仍作为重点参考墙区。

解释：

如果主力愿意把价格推进并站稳在 0.0022 上方，说明这片压力带开始被消化。此时更像逼退空头或逼空，而不是普通反弹。

### 墙下派发 / 转弱观察

触发倾向：

- AKE 必须先接近或试探过 0.0020 压力带，不能离压力带还远就误判失败；
- 随后价格跌回 0.00195 下方；
- 30MIN/1H/2H 中，价格和 CVD 都至少两个窗口转弱；
- 成交量出现活跃但价格无法突破。

解释：

如果主力不吃墙，墙本身就不是“必然逼空目标”，而可能变成出货天花板。

## 系统实现备注

Binance 合约 depth API 只返回有限档位。当前价离 0.0020 较远时，1000 档公开深度可能覆盖不到 0.0020–0.0023，因此系统区分两类数据：

- 实时可见盘口墙：API 深度覆盖到墙区时使用；
- 用户观察参考墙区：API 暂时覆盖不到时，只作为重点价位参考，不假装为实时挂单。

## 深度档位限制

- BN U 本位合约公开 REST 深度接口最大为 1000 档，系统已使用最大档。
- BN 现货公开 REST 深度接口最大为 5000 档，系统会尝试作为辅助卖压参考；如果该币没有 BN 现货或接口不返回，则不参与判断。
- 用户在 BN 网页端看到的更远深度，可能来自网页聚合档位或内部前端数据，不一定能通过公开合约 REST API 直接拿到。
