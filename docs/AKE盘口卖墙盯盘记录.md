# AKE 盘口卖墙盯盘记录

## 用户新思路

AKE 在盘口上方出现重点卖墙/压力带：

- 0.0020 附近：约 1.1M AKE 卖单
- 0.0021 附近：约 1.2M AKE 卖单
- 0.0022 附近：约 1.0M AKE 卖单
- 重点压力带：0.0020–0.0023

用户的两个假设：

1. 如果主力主动把价格拉到 0.0020–0.0022 以上，说明愿意吃掉或逼退这部分压力，后续可能继续拉盘，把上方做空/卖压爆掉或逼走。
2. 如果价格一直站不上 0.0020，则可能在 0.0020 下方开始派发，随后转为下跌。

## 我的补充判断

盘口挂单不能直接等同真实空单，因为挂单可以撤，也可能是诱导盘口。但它可以作为“主力是否愿意吃墙”的观察线索。

因此 AKE 卖墙盯盘必须结合：

- 是否站上 0.0020；
- 是否进一步站上 0.0022；
- 突破时 CVD 是否继续上涨；
- 持仓是否没有同步塌掉；
- 成交量是否放大；
- 基差是否继续正向打开；
- 是否出现空头爆仓跟随。

## 盯盘规则

### LARK 推送降噪规则

AKE 盘口墙属于“方向确认前的观察信息”，不能因为正资费、正基差、普通多头结构就频繁推送。

从 2026-07-18 起，AKE LARK 推送只保留三类更明确的信号：

- 卖墙突破：价格站上 0.0022，且 CVD/持仓配合；
- 墙下派发：价格站不上 0.0020，且量能活跃、价格或 CVD 转弱；
- 出货警报：放量、资金费率转负、基差明显打开同时出现。

普通“正资费 + 正基差 + 合约量大”的多头观察，只在网页里显示，不再自动推送 LARK。

AKE 不设置固定冷却时间。机会来了必须立刻通知。

推送逻辑改为“状态变化即时推送”：

- 试探 0.0020；
- 进入 0.0020–0.0022 卖墙区：必须提醒；
- 突破 0.0022 上沿：必须提醒，归类为强势上涨/逼空走势；
- 插上 0.0020 后快速回踩：不直接视为失败，优先按诱空/试盘观察；
- 反复试探 0.0020 失败后跌回 0.00195 下方并转弱。

同一个完全相同的状态与价格桶不重复推；但只要进入新状态，或价格推进到新的关键区间，就必须立刻推送。

### 当前假设：0.002 插针回踩可能是诱空

用户思路（待验证，不作为固定知识点）：

AKE 向上插针到 0.002 以上后立刻回调，不应直接理解为上攻失败。因为 0.002 是盘口重点墙区下沿，一旦插上去，很多空单挂单和追空心理会被激活。主力不一定马上砸盘，反而可能先诱导空单进场，随后继续上涨。

盯盘验证方式：

- 只要最近高点/压力位触及过 0.0020；
- 当前回踩仍在 0.00192–0.0020 区间；
- 持仓没有塌、CVD 没有连续恶化，或量能没有形成放量下杀；
- 临时标记为“插针回踩 / 诱空假设观察”，继续盯后面是否重新站上 0.0020。

验证成功条件：

- 回踩后重新站上 0.0020；
- 进一步进入 0.0020–0.0022 区间；
- 若站上 0.0022，则说明用户假设更可能成立，归类为强势上涨/逼空走势。

证伪/降级条件：

- 跌破 0.00190；
- 价格、CVD 连续转弱；
- 持仓不再增加甚至同步转弱；
- 则把它从诱空假设降级为墙下失败。

### 卖墙突破 / 逼空观察

触发倾向：

- AKE 价格进入 0.0020–0.0022 区间，就已经进入卖墙战场，必须推送；
- AKE 价格进一步站上 0.0022，归类为强势上涨/逼空走势，必须推送；
- CVD、持仓、量能只作为推送中的辅助判断，不再作为触发提醒的必要条件；
- 0.0020–0.0023 压力带仍作为重点参考墙区。

解释：

如果主力愿意把价格推进到 0.0020 上方，说明已经开始进入压力区，必须提醒。0.0022 是上沿强势确认，站上后按强势上涨/逼空走势提醒。

### 墙下派发 / 转弱观察

触发倾向：

- AKE 必须先接近或试探过 0.0020 压力带，不能离压力带还远就误判失败；
- 随后价格跌回 0.00195 下方；
- 30MIN/1H/2H 中，价格和 CVD 都至少两个窗口转弱；
- 成交量出现活跃但价格无法突破。

解释：

如果主力不吃墙，墙本身就不是“必然逼空目标”，而可能变成出货天花板。

## 系统实现备注

Binance 合约 depth API 只返回有限档位。当前价离 0.0020 较远时，1000 档公开深度可能覆盖不到 0.0020–0.0023，因此系统区分两类数据：

- 实时可见盘口墙：API 深度覆盖到墙区时使用；
- 用户观察参考墙区：API 暂时覆盖不到时，只作为重点价位参考，不假装为实时挂单。

## 深度档位限制

- BN U 本位合约公开 REST 深度接口最大为 1000 档，系统已使用最大档。
- BN 现货公开 REST 深度接口最大为 5000 档，系统会尝试作为辅助卖压参考；如果该币没有 BN 现货或接口不返回，则不参与判断。
- 用户在 BN 网页端看到的更远深度，可能来自网页聚合档位或内部前端数据，不一定能通过公开合约 REST API 直接拿到。
