# 云端部署准备

当前本地版和未来云端版共用同一套 Flask、MySQL 与后台任务代码。

## 已就绪的内容

- `Dockerfile`：封装网站与后台采集服务。
- `docker-compose.yml`：同时启动网站与 MySQL，并配置服务自动重启。
- `wsgi.py`：云端进程启动时只启动一组后台采集与数据完整性任务。
- `.env.cloud.example`：云端 MySQL 连接与密钥配置模板。

## 数据完整性

- 服务启动后的低优先级后台任务会定期检查最近 7 天的 5 分钟合约价格点；本地 SQLite 环境会自动降频运行，避免与 5 秒实时行情快照竞争写入。
- 缺少趋势关键时间点或有效点位不足时，自动低频分批回补 Binance 历史 K 线。
- 价格历史只滚动保留 8 天；交易所暂时不可用时保留已有数据，并在下一轮继续检查与回补。
- `GET /api/data-integrity` 可读取当前行情快照时间、已完整币种数量与待补币种列表，供本地或云端巡检使用。

## 未来上线时

1. 准备一台 24 小时在线的云服务器，并安装 Docker。
2. 将项目上传到服务器，复制 `.env.cloud.example` 为 `.env`，替换密码和密钥。
3. 使用 `docker compose up -d --build` 启动。
4. 配置反向代理、HTTPS、防火墙、数据库备份与访问控制。

云端上线前应先与项目负责人确认域名、访问方式、数据库备份周期及告警通知渠道。
