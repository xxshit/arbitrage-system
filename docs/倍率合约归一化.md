# 倍率合约与底层资产归一化

记录时间：2026-07-18

## 核心问题

有些交易所合约会用倍率包装同一个底层资产，例如：

- `1000XEC/USDT` 合约
- `XEC/USDT` 现货

它们不是两个完全独立的币，而是同一个底层资产在不同交易产品里的单位表达。

## 系统处理规则

1. 不允许简单把所有 `1000` 前缀都删掉，因为有些币名本身可能带数字。
2. 必须维护一个明确的映射表，例如：
   - `1000XECUSDT -> XECUSDT`
   - 合约价格单位 = 1000 × 现货价格单位
   - 归一化底层资产 = `XEC/USDT`
3. 现多期空中，允许用 `XEC` 现货按倍率换算后，对接 `1000XEC` Binance 合约。
4. 期多期空中，继续保留真实合约名，例如 `1000XEC/USDT`。
5. 涨跌幅榜中，同一底层资产只显示一次：
   - 上涨榜保留涨幅更高的那条；
   - 下跌榜保留跌幅更深的那条。

## 后续扩展

后续遇到类似币种时，不要盲目自动合并。优先确认：

- 交易所元数据；
- 价格倍率是否稳定对应；
- 官网链接、项目名称、合约地址或官方资料是否一致；
- 多交易所命名是否指向同一底层资产。

确认后再加入映射表，避免把不同资产误合并。
