# 套利策略知识

## 策略范围

### 1. 跨交易所合约套利

同一标的在不同交易所的永续或交割合约存在可交易价差时，低价端开多、高价端开空。核心是两边名义价值尽量匹配，收益来自价差收敛，成本包含双边手续费、滑点、资金费率与资金划转成本。

### 2. 现货做多 / 合约做空（Delta Neutral）

买入现货，同时等额卖出永续合约。市场方向风险被大致对冲；若永续合约资金费率为正，空头通常收取资金费率。应关注基差变化、合约强平风险和现货/合约数量匹配。

### 3. 资金费率套利

以资金费率收益为核心，通常在正费率时持有现货多头和永续空头，在负费率时采取反向的可行对冲组合。资金费率不是固定收益，必须在结算前再次核验费率预测与仓位规则。

## 预估净收益

当前看板示例：

```text
预估净收益率 = 卖出价格 / 买入价格 - 1 - 预估双边成本
```

## 现多期空：盘口与价差口径

现阶段的现货多头交易所为 **Binance、Gate、Bitget**；合约空头交易所固定为 **Binance USDⓈ-M 永续合约**。界面中现货多头始终显示在左侧，Binance 合约空头始终显示在右侧。

为确保计算基于可成交价格：

- 现货开多使用现货 **卖一价（ask 1）**；现货平多使用现货 **买一价（bid 1）**。
- 合约开空使用合约 **买一价（bid 1）**；合约平空使用合约 **卖一价（ask 1）**。

```text
开差% = (做空交易所买一价 - 做多交易所卖一价) / 做多交易所卖一价 × 100%
平差% = (做空交易所卖一价 - 做多交易所买一价) / 做多交易所买一价 × 100%
```

合约基差不属于现货交易所。系统统一以 Binance 合约的标记价和 Binance 现货指数计算：

```text
合约基差% = (合约标记价 - 现货指数) / 现货指数 × 100%
```

因此，开差、平差用于比较现货与合约的可成交价；合约基差独立反映 Binance 合约相对于标的现货指数的偏离。

## 多周期资费统计口径

- 当前：Binance 当前公布的资金费率。
- 上期：最近一笔已经结算的历史资金费率。
- 24H、3D、7D、30D：按 **北京时间（UTC+8）自然日** 汇总的已结算资金费率。
- 单日窗口为 `00:00:00（含）` 至次日 `00:00:00（不含）`；每一笔结算只按其结算时间归入一个自然日，避免跨日重复计算或遗漏。
- 24H 为当日截至当前时刻的已结算累计；3D、7D、30D 均包含当日及此前对应数量的自然日。

## 现多期空报警规则

### 合约拉升报警

满足以下两个条件时触发：

```text
开差 > 1.0%
abs(开差 - 平差) < 0.6%
```

该规则用于筛选合约快速拉升、而现货与合约两端盘口差距仍相对一致的情况。

### 现货拉升报警

满足以下两个条件时触发：

```text
开差 < -1.0%
abs(开差 - 平差) < 0.6%
```

该规则用于筛选现货快速拉升导致开差转为显著负值的情况。负基差仍走下方的连续两次采样确认逻辑，避免把单次插针视为有效信号。

### 基差异常与 MySQL 轨迹

- 当合约基差大于等于 `+1.0%` 或小于等于 `-1.0%` 时，先进入候选状态。
- 只有连续两次采样均处于同方向阈值之外才确认，单次插针不会报警或写入轨迹。
- 确认后创建一条 MySQL 基差追踪记录，在 `1.0%` 留下起始日志；绝对基差每扩大 `0.2%`，新增一条扩展日志。
- 追踪记录持续更新最大基差和最大值出现时间。基差回落到绝对值小于 `0.8%` 后结束本轮追踪，形成滞回以避免阈值附近反复开关。
- 全局 TIPS 浮窗展示最新确认报警，可在任意页面展开或收起；报警中心按交易对保留一张最新、最强的详细卡片。
- 报警中心与 TIPS 按交易对展示最新状态：同一交易对的新报警覆盖旧展示，避免重复卡片造成漏读。报警标签必须携带策略来源，例如“现多期空 · 合约拉升”，便于后续增加双合约等策略。
- 报警中心采用峰值覆盖：同一交易对仅在报警强度继续扩大时才更新卡片；拉升/下挫类以开差绝对值判断，基差异动以基差绝对值判断。卡片仅保留开差峰值与触发时间，方便快速确认入场时的信号强度。
- 实时 TIPS、报警卡片与声音采用 0.2% 扩大门槛：同币种第一次确认报警会提示；此后仅当开差绝对值或基差绝对值较上一次已提示值扩大至少 0.2% 时才覆盖并再次提示，避免 5 秒轮询造成重复干扰。

生产计算应至少拆分：

```text
净收益 = 价差收益 + 预计资金费率收益
       - 开平仓手续费 - 滑点 - 借币/资金成本 - 提币与划转成本 - 预留风险成本
```

## 不可省略的风控

- 不把“显示价差”当作可成交收益：必须按盘口深度计算可成交均价。
- 下单前核验最小下单量、价格精度、合约面值、杠杆、保证金模式与最大仓位。
- 设置单策略、单交易所、单币种和总敞口限额。
- 处理单腿成交：优先自动对冲，超时后报警或平仓，不允许裸露方向风险无限延续。
- 记录每笔订单、成交、资金费率、手续费和异常，便于对账。
- API 密钥必须最小权限、IP 白名单、加密保存；禁止授予提现权限。

> 本项目用于研究和监控，任何真实下单功能上线前须进行独立风控评审与小额验证。

## 报警确认与展示约定

- 合约拉升与现货拉升报警必须和基差报警一样，连续两次行情采样在同一方向、同一交易所路径上满足阈值后才确认；单次触及后回落按插针处理，不写入报警中心、不弹出 TIPS。
- 报警中心与全局 TIPS 均须显示策略类型和交易所路径，并以交易所图标辅助识别。现多期空的路径统一为“做多交易所现货 → Binance 合约”；未来期多期空报警使用独立的“期多期空”策略标签与对应合约路径。

## 期多期空：首批交易路径与口径

### 期多期空报警口径

- 期多期空采用与现多期空相同的插针防护：同一交易所合约路径连续两次（5 秒采样）仍满足条件，才写入报警中心与实时 TIPS。
- 开差报警：路径开差的绝对值达到 `1.0%` 起触发；后续仅在绝对值相对该币种、该策略最近报警扩大至少 `0.2%` 时覆盖报警并再次提示。
- 基差报警：只使用 Binance 合约基差判断；仅包含 Binance 的 Bybit→Binance、OKX→Binance 路径可参与该项判断。Binance 基差绝对值达到 `1.0%` 起触发，后续同样以 `0.2%` 的绝对值扩大量更新。
- Bybit→OKX 路径没有 Binance 合约端，因此保留开差报警，不做 Binance 基差报警。
- 报警中心按“策略 + 币种”保留最新一条记录，并显示期多期空标签及对应的多端、空端合约路径，避免与现多期空同名币种混淆。

### 基差打开轨迹展示

- “打开”定义为同方向基差绝对值首次连续两次确认越过 `1.0%` 的时刻；记录打开时间与当时基差值。
- 基差绝对值此后每扩大 `0.2%`，持久化一条扩展日志，同时令打开次数加一；打开次数等于该轮轨迹的持久化日志数。
- 每轮轨迹持续保存近期最大基差、最大值出现时间；基差回落至绝对值小于 `0.8%` 后结束本轮。
- 现多期空与期多期空均在表格末端显示“基差打开”入口；悬浮显示首次越阈、近期最大值与时间、累计打开次数。期多期空的轨迹只以 Binance 合约基差建立。

- 首批固定路径为：Bybit 合约做多 → Binance 合约做空、OKX 合约做多 → Binance 合约做空、Bybit 合约做多 → OKX 合约做空。界面中做多方固定在左、做空方固定在右，并使用交易所图标。
- 期多期空开差使用“做空合约买一价”与“做多合约卖一价”计算；平差使用“做空合约卖一价”与“做多合约买一价”计算，均基于可成交的第一档盘口。
- 每一端合约均独立显示其相对自身指数价格的合约基差；资金费差按“做空端资金费率 − 做多端资金费率”显示，正值代表该对冲方向的预期资金费净流入（不含手续费、滑点与执行风险）。
- 趋势灯与涨跌幅按多端、空端分别采集和展示，避免用其中一边的走势替代整组对冲路径。
- 期多期空页面的趋势灯与涨跌幅统一以 Binance 合约作为币种级参考，每个币种仅显示一次；合约基差不再集中展示，而是紧贴各套利路径中对应交易所合约的名称显示。双方交易所资费各自显示，结算周期紧跟在该侧资费右方。
- 期多期空的“下期资费差”是对冲仓位在下一次实际结算事件的净资金费收益：做多端收益为“资费率取负”，做空端收益为“资费率保持原符号”。两端下一结算时间相同才合并相加；若下一次仅有一端先结算，则只计入先结算的一端，并标明“仅多端结算”或“仅空端结算”及时间。正值代表该次结算预期净收取，负值代表预期净支付。
- 下期结算时间统一按资金费周期的上海时区整点网格推算，不直接用各交易所可能锚点不一致的 `nextFundingTime` 字段：例如当前处于 00:00—04:00，4H 的下一档为 04:00、8H 的下一档为 08:00；双方周期相同即视为同一档同时结算。
- 期多期空的“共指”用于衡量套利路径两端合约指数**成分来源与权重**的交集，不能用两个指数价格的接近程度替代。先统一同义来源名称（如 `binance_future` 与 `Binance_Futures`），对每个共同来源取两边权重的较小值，再求和得到共指百分比；表格只显示该百分比。悬浮提示必须列出两家交易所各自的真实指数成分与权重，并以进度条显示共同成分权重。若交易所不存在该币指数或尚未完成后台同步，明确显示不可用或待同步。
- 流动性展示统一使用美元名义的 24H 成交额与合约持仓量：现多期空的现货端只显示该现货交易所的成交额，Binance 合约显示成交额与持仓量；期多期空的多、空两端合约都显示各自成交额与持仓量。`10M USD` 为风险分界，低于阈值时必须有底框警告，按 `5M—10M`、`1M—5M`、`<1M` 依次加深颜色。
- 美股 RWA/TradFi 合约不进入现多期空与期多期空的搜索、排序、分页与展示。识别以 Binance 合约元数据的 `TRADIFI_PERPETUAL`、`EQUITY` 或 `TradFi` 分类为准，而不是手工维护股票代码名单；后台快照保留这类数据，供后续独立的美股代币索引使用。
- 期多期空的合约基差排序必须分别支持 BN、BY、OK 三家交易所，每个按钮独立支持正反排序。共指悬浮面板按成分来源逐条对照：每条共用一个轨道，上层显示左侧多端交易所权重、下层显示右侧空端交易所权重，使共同与独有权重一目了然。
# 展示口径补充

- 现货市场不产生持仓量。所有页面（包括套利思维和后续详情）中，现货腿只展示交易量；持仓量仅用于永续合约腿的流动性与 ADL 风险判断。

## 趋势研究的指标口径

- 趋势分析需要联动价格、成交量、持仓量（OI）、多空比、主动成交方向（CVD）与资金费率；其中任何单一指标都不能直接给出方向结论。
- OI 增减只代表未平仓合约总量变化，不代表净多或净空；新增合约同时存在多空两端。结合价格与 CVD 后，才能将其归类为新增仓位推动、回补或平仓释放的候选情景。
- 多空比必须标明数据口径。全市场账户多空比与头部交易者账户/仓位多空比可用于比较群体倾向，但“主力”“散户”只能作为待验证假设。
- CVD 以主动买入成交额减主动卖出成交额的累计值表示；量价与 CVD 背离是观察信号，需由 OI、成交量与后续价格确认。
- 犄型走势（偏多）定义：持仓量上涨，同时多空人数比呈对称下跌，候选解释为主力做多、散户做空；若 CVD 同步上涨，说明主动买入也在增强，对“主力布置多单”的假设更有力。该结构仍必须通过后续价格延续、成交量变化和关键位得失验证，不能只凭单个截图或单次数据定性。
- AKE 盯盘口径：以用户入场点 `0.00085` 为成本锚点；后台每 5 分钟检查一次。只有近 30M、1H、2H 同时满足价格转强、持仓增加、多空人数比下降、CVD 上涨，才允许走势分析机器人推送；否则只在“思路分析”页面更新本地数据，不打扰。
- 大涨前回调诱导平多：只能作为待验证假设。若回调期间价格守住关键支撑、持仓不快速塌陷、CVD 不持续转负，并且后续重新放量上破，则更支持“洗出短线多头后继续拉升”；若回调时 CVD 持续转负、OI 快速下降或跌破结构支撑，则更偏向获利了结/派发，不能按洗盘处理。
- 已止盈后的二次机会盯盘：不能继续用老仓位的盈利惯性判断。多头二次机会必须重新满足犄型共振；空头机会必须满足价格走弱/跌破支撑、持仓扩张、多空人数比回升、CVD 转负的主动空头共振。单独价格波动不构成推送理由。
- 多倍拉升后的止盈复盘：若币种已从底部拉升数倍，但上涨过程中没有明显放大量、持仓没有快速掉落，则不能把短暂小回调直接视为主力出货；它可能是诱导短线多头平仓的洗盘。真正出货更应看组合证据：狂暴放量、持仓快速掉落或异常扩张后价格滞涨、CVD 明显背离、合约基差/套利价差异常拉开、资金费快速转负或显著恶化。条件越多共振，出货判断越可靠。
- 高涨幅山寨的涨势反转提醒：US、AKE 这类在弱势大盘里逆势大涨的币，不因“涨多了”直接做空；必须等待反转证据。优先级从高到低为：跌破近端结构后反抽失败、狂暴放量冲高回落或砸盘、CVD 由强转弱并持续为负、持仓快速掉落或异常扩张后价格滞涨、资金费转负或快速恶化、合约基差/套利价差异常拉开。至少价格结构 + CVD + 资金/持仓/基差价差之一共振，才触发走势机器人。
- US/AKE 出货三件套强提醒：若出现大量放量、资金费转负、基差打开三项同时成立，必须通过走势分析机器人提醒，并附带分析。默认阈值：近 30M/1H/2H 任一窗口成交额达到前 10 根 30M 均值的 `2.5x` 以上，Binance 资金费率 `< 0`，Binance 合约基差绝对值 `>= 1%`。该信号视为主力出货/强制换手预警，不等于立即追空；后续要继续看是否跌破支撑、CVD 是否持续转负、反抽是否无力。
- 出货换手的资金费周期信号：若币价已大幅拉升，随后出现放量拉基差、资金费极端化，并且结算周期从 `4H/8H` 缩短为 `1H`，要视为高优先级换手/出货观察点。若后续价格缩量下跌、反弹时仍维持 1H 结算和极端负资费，则反弹更可能是诱多；但仍需用价格是否重新站回关键位、CVD 是否转正、基差是否修复来确认。

## 报警规则：短时速变

- 除绝对阈值报警外，只监控 30 秒内开差与基差绝对值的短时扩大：扩大至少 0.5% 为“速变”，扩大至少 1.0% 为“强速变”，视为疑似放量操盘的速变信号。
- 速变报警仍必须连续两次 5 秒采样确认，且使用同一交易所路径与同一指标；单点报价异常不会写入报警中心。
- 现多期空监控该现货交易所对 Binance 合约的开差与 Binance 合约基差；期多期空监控每条合约路径的开差及该路径中的 Binance 基差。
- 速变报警触发后进入 30 分钟窗口：同一币种、同一路径、同一指标在窗口内只有绝对值比上一次报警值继续扩大至少 0.2% 才允许再次报警，并刷新窗口起始时间；如果 30 分钟内没有继续扩大，窗口结束后才重新执行速变报警逻辑。
- 速变信号用于更早发现异常，不替代绝对阈值、盘口复核、成交量与流动性判断。
- 所有报警写入前必须再做最终扩大量校验：同一币种、同一策略、同一报警类型，在最近 30 分钟内以已写入报警的峰值为准，当前用于该类型的指标绝对值必须比该峰值扩大至少 `0.2%`。不同路径不能绕过这个峰值限制。例如 HOME 开差从 `-2.3%` 回落到 `-2.2%`，即使仍满足阈值，也不得再次写入、不得再次触发声音。
