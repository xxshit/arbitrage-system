# 趋势研判日志

本文件记录趋势研究中的原始假设、数据证据、后续结果与复盘。每次研判均须以时间戳追加，不删除已验证或被证伪的结论；若判断错误，保留原判断并写明修正原因。

## 研究原则

1. 先描述数据，再给出假设，最后以预先约定的观察窗口验证；不把单次指标变化直接当作交易结论。
2. 持仓量（OI）仅说明未平仓合约总量的变化，不单独说明新增的是多头还是空头。方向需由价格、主动成交方向（CVD）、成交量及多空比共同佐证。
3. 多空比必须注明口径：全市场账户多空比、头部账户多空比、头部仓位多空比的含义不同。它们只能作为群体仓位倾向的代理，不可直接等同于“主力”或“散户”。
4. CVD 用主动买入成交额减主动卖出成交额的累计值衡量。上涨而 CVD 走弱，或下跌而 CVD 走强，属于背离候选信号，必须结合成交量、OI 与后续价格确认。
5. 每次研判默认记录 4 小时、12 小时、24 小时三个验证窗口；结论使用“偏多/偏空/中性/等待确认”，不使用确定性预测。

## 指标组合速查

| 价格 | OI | 成交量 / CVD | 初步解读 | 需要防范 |
|---|---:|---|---|---|
| 上涨 | 上升 | 放量、CVD 上升 | 新仓推动的上涨，偏多延续候选 | 高位追多拥挤、资金费过热 |
| 上涨 | 下降 | CVD 不强或回落 | 空头回补推动的上涨候选 | 回补结束后冲高回落 |
| 下跌 | 上升 | 放量、CVD 下降 | 新仓推动的下跌，偏空延续候选 | 恐慌末端、急跌反抽 |
| 下跌 | 下降 | CVD 走弱 | 多头平仓/清算推动的下跌候选 | 抛压释放后的技术反弹 |
| 横盘 | 上升 | 成交量放大、CVD 单边 | 仓位积累，等待价格选择方向 | 双向加仓，不能预判方向 |

- 我的观点：这确实是“价格上行 + OI 上行 + CVD 上行”的趋势延续候选结构，至少能说明拉升阶段存在主动买入与新增杠杆仓位，不能简单视为纯粹的空头回补。人数多空比降至低位，与用户所说的“市场账户偏空”一致；若头部仓位多空比同期走高，才更能支持“头部偏多、市场账户偏空”的犄型假设。
- 关键保留：
  1. OI 增加只代表新增未平仓合约，不能独立证明新增多头；每一份新合约同时存在多空双方。
  2. CVD 高位走平而价格继续缓慢抬升，短线既可能是买盘吸收后的控盘上行，也可能是主动买入衰减后的派发前兆；需观察价格创新高时 CVD 是否再创新高。
  3. 正资金费并不天然等于“主力希望散户做空”。正费率表示多头向空头支付资金费，通常反映永续多头相对拥挤或现货/合约结构偏强；若人数比低而费率仍正，说明不同口径存在分歧，正是需要继续观察的价值所在。
  4. 初始突破段成交量明显放大，之后量能回落属于正常的突破后整理，也可能意味着承接不足。不能仅凭“没有再次巨量”认定没有出货。
- 当前结论：偏多，但处于高波动突破后的等待确认阶段；不追认“主力控盘”，优先观察结构是否延续。
- 验证条件：
  - 偏多确认：30 分钟收盘持续守住突破后的平台低点；OI 不出现快速塌陷；价格新高时 CVD 再创新高或至少不明显背离；人数多空比低位而不发生极端反转。
  - 多头失效/转谨慎：价格新高但 CVD 连续走低，OI 同步快速下降；或高位放出显著大成交量却无法继续上破，随后跌回突破平台。
  - 风险观察：资金费率若快速抬升而人数比也由低位急升，需防范多头拥挤后的反向清算。

### AKE/USDT / 2026-07-18 / 犄角延续型规则修正

- 用户观察：AKE 截图中红框部分仍然是明显的犄角结构。虽然持仓从峰值回落了一些，但价格持续抬高，CVD 继续向上，多空人数比长期压在低位，说明最初的“持仓上升 + 人数比对称下跌”主升结构并没有完全失效。

- 新分类：犄角走势拆成两类：
  1. 启动型犄角：观察窗口内价格上涨、持仓上涨、多空人数比下跌，CVD 上涨更佳。
  2. 延续型犄角：窗口内不强制要求持仓首尾继续上涨，但必须满足“前期持仓爆发后仍保留大部分仓位、价格继续抬高、多空人数比维持低位或继续下压、CVD 继续抬高或不明显背离”。
- 延续型候选条件建议：
  - 当前 OI 至少仍高于启动前 OI 的 1.5 倍，或至少保留峰值 OI 的 60%-70%。
  - 当前多空人数比处于最近窗口低位区，例如低于窗口中位数，或低于 0.8；若仍在继续下压则加分。
  - 价格相对启动前仍明显抬高，并保持高低点上移；不要求每个小窗口都涨。
  - CVD 从启动点到当前仍为正向累积，若价格新高时 CVD 也新高则大幅加分；若价格新高但 CVD 背离则降级为观察。
  - 资金费、基差若仍为正，说明合约端仍在被抬高，不能轻易按出货看空；若资金费转负、基差异常打开、放量冲高回落，则转入出货观察。

#### 2026-07-18 二次修正：延续型必须看币本位 OI 结构线

- 用户指出：ESPORTS 虽然被系统打成高分延续，但 CoinGlass 图中当前持仓已经低于近期持仓结构中线，甚至低于结构底线；这种状态不能叫犄角延续。
- 错误原因：系统最初使用 Binance `sumOpenInterestValue`（美元名义持仓）计算 OI 延续。价格上涨会把美元名义持仓放大，导致币本位持仓已经下降时，美元持仓仍看起来不弱。这和 CoinGlass “Coins open” 持仓 K 线口径不一致。
- 修正规则：
  - 犄角延续的结构判断必须使用 Binance `sumOpenInterest`（币本位持仓数量），不是 `sumOpenInterestValue`。
  - 先找前期 OI 爆发后的结构区间，再计算结构底线与结构中线。
  - 当前币本位 OI 跌破结构底线：直接剔除，不得进入犄角延续榜。
  - 当前币本位 OI 位于结构底线和中线之间：只能低分观察，不能高分。
  - 当前币本位 OI 位于结构中线之上，同时人数比继续下降、CVD 正向确认，才允许高分。
  - 人数比不再下降或 CVD 不确认时，不进入延续榜。
- 复核结果：按新规则重扫后，ESPORTS 被剔除；当日犄角延续榜为空。空榜是可接受结果，说明当前没有干净的延续型犄角，不应为了有内容而降低标准。
